# Mindwtr Unreleased

Changes collected after `v1.1.0` and before the next version tag.

## Highlights

_None yet._

## Full Change List

- Switched mobile local storage to a native SQLite engine (op-sqlite): SQL now executes on a dedicated native thread with a faster call path, the first step of moving storage and sync work off the UI thread on large libraries. Existing data is picked up in place; no migration.
- Finishing or editing a task on mobile no longer waits for the full-library backup snapshot: saves complete after the database write, and the backup and widget refresh land coalesced in the background (bursts of saves now write one backup instead of one each — a single save had spent up to 12 seconds on this with large libraries). Reminder rescheduling also coalesces the several store updates a sync cycle makes into one scan. (#766)
- Friendlier first run for people new to GTD: the welcome screen and the Getting Started project now use plain language, a new starter task shows how to hide task-editor fields you don't use, and an empty Focus view now explains how tasks get there (with capture guidance when the app has no tasks yet).
- The Getting Started project, its tutorial tasks, and the sample inbox items are now created in your app language (17 languages). Re-adding the content after switching languages repairs it in place instead of duplicating it. The first-run welcome screen is translated too.
- The AI assistant now works with newer Claude models (Sonnet 5, Opus 4.7/4.8): requests no longer send a temperature these models reject, and the thinking toggle uses their adaptive thinking mode. Enabling thinking on older Claude models no longer fails either. (#857)
- Android text no longer loses its last letter or line on Android 15/16 at large font sizes ("Off" showing as "Of", settings descriptions ending mid-sentence): the app now pins the pre-Android-15 text drawing behavior so drawn text always matches its measured space. Settings rows and the welcome screen also adapt their layout at large font scales instead of squeezing text into narrow columns. (#632)
- Reopening into Calendar, Archive, or another non-tab screen now keeps the main tabs in the navigation stack, so the iOS header Back button remains available and Android Back returns to the app instead of closing it. (#842)
- Desktop settings toggles no longer get squashed into ovals when the window is narrow — the switch now keeps its shape and the label text wraps instead. (#858)
- Desktop keyboard shortcuts no longer act on the task list behind an open dialog: with search, quick add, or a prompt open, a stray Enter or action key could complete or open a task in the background list (reported as an accidental Done from inside search). Keys pressed inside the search dialog outside its input now also keep driving the search — arrows and Enter navigate results, and typing returns to the query field.
- Updated bundled dependencies to pick up upstream security fixes (hono, undici, tar, vite, js-yaml, Babel on the JS side; plist/quick-xml, quinn-proto, anyhow in the desktop app).
- The "What's the next action?" prompt (shown after finishing a project's last action) now understands quick-add syntax, so `Chase the reply /waiting %Bob` creates the follow-up as a Waiting For task directly — no need to create it as a next action and edit it afterwards. (#859)
- The Focus toolbar now matches every other list: its buttons and the grouping dropdown share one height and shape, the grouping dropdown carries the same `GROUP` label as elsewhere, and its text is no longer clipped at the bottom. (#861)
- Focus now keeps future-start tasks hidden until their start day without showing a Show/Hide notice or control, on desktop and mobile. The separate reveal control in the Next list is unchanged.
- Desktop lists now set a task's status straight from the keyboard: press `s` then a letter — `si` Inbox, `sn` Next, `sw` Waiting, `ss` Someday, `sd` Done, `sa` Archived — with an undo toast after the change. The chord works in all three keybinding styles, and `Insert` jumps to the add-task input (or opens quick add). (#860)
- Desktop can now capture email into the Inbox (Settings → Integrations → Email capture): point it at an IMAP folder with an app password (Gmail, iCloud Mail, Fastmail, or any standard IMAP server) and, while Mindwtr is running, each email in that folder becomes an Inbox task — subject as the title, sender and body as the description. Mail is read-only: nothing is modified, moved, or deleted, the folder is created for you if missing, and already-imported messages are remembered so nothing imports twice. (#35)
- The desktop "switch to Area" chord is now forgiving: keeping Shift held from the `A` into the number (which types `!`, `@`, …) still switches areas, pressing Shift mid-chord no longer cancels it, and with Caps Lock on a plain `a` opens quick add instead of silently arming the chord. The shortcut help sheet now spells the chord as `Shift+A 1-9` so it reads as typed. (#865)
- Adding a task to a project during the weekly review is no longer title-only: the prompt understands the full quick-add syntax (`Chase quote @phone %Bob /waiting`), and a new "Save & edit" button creates the task and opens it in the editor — without leaving the review or losing your place. Works on desktop and mobile.
- The desktop Proxy URL setting now applies to all sync traffic. Self-hosted cloud, WebDAV, and Dropbox connection requests had been going out directly, ignoring the saved proxy (only reachable via `HTTP_PROXY`/`HTTPS_PROXY` env vars as a workaround); they now route through the configured proxy, taking effect on save with no restart. An invalid proxy URL is rejected when saving instead of being silently skipped. (#864)
- Completing a recurring chore no longer clutters Next actions with the occurrence you cannot start yet: a repeat that carries only a due date (a bimonthly "descale the kettle") now stays out of Next actions until it comes due, exactly as it already did in Focus. The deferred occurrences are counted by the "hidden (future start)" notice above the list, with a **Show** toggle to reveal them. The Android daily review defers them the same way. (#867)
- "Repeat after completion" tasks that also pick weekdays (or a day of the month) now honor the **Repeat every** interval. An "every 2 weeks on Sunday" chore completed on a Tuesday used to come back the very next Sunday — and the calendar preview showed the same too-early date — no matter what interval was set; it now returns on the Sunday of the second week after completion. Fixed-schedule repeats are unaffected. (#867)
- Desktop keyboard shortcuts no longer go dead after deleting a task from its open editor (for example Quick Add → Save & edit → Delete): the app was still treating the removed task as being edited and kept suppressing every shortcut until another edit was opened by mouse. (#870)
- The Trash now supports bulk actions on desktop and mobile, completing the set alongside Completed and Archived: tap **Select** to pick individual tasks and projects (or **Select all**), then restore them to the Inbox or delete them permanently in one step — permanent deletion still asks for confirmation. (#863)
- The desktop Add Task dialog is now fully keyboard-driven: `Ctrl+Enter` (`⌘Enter` on macOS) saves and opens the task in the editor, and `Shift+Enter` saves and keeps the dialog open for the next capture — batch entry without touching the mouse (the Save buttons show these shortcuts as tooltips). Tab no longer stops on the dialog backdrop or the `Esc` chip, and the syntax help line now sets the typable tokens (`/start:`, `@context`, `+Project`, …) apart from their `<placeholder>` parts in every language. (#869)
- The weekly review's final screen now ends with a short, calm status summary instead of only a congratulation: whether your Inbox is empty, whether every active project has a next action, and how many Waiting For items have sat untouched for more than two weeks. Each line mirrors what the matching review step showed, and the good/needs-attention rows read at a glance without scores or noise. Works on desktop and mobile. (#871)
- Setting a start date on an Inbox item now clarifies it to Next — the same shortcut starring already took. Deciding *when* you can act on something is a clarify decision, and before this an Inbox item with a start date was invisible in Focus (Focus only lists actionable tasks), which read as the date being ignored. Someday and Waiting items keep their status when dated (a dated someday is a tickler, a dated waiting-for a follow-up reminder), picking a status in the same edit wins, and a date arriving still never changes status on its own. Works everywhere tasks are edited or captured: editors, quick-add `/start:`, review defer, and the MCP/CLI/API surfaces.
- Desktop on older macOS (WebKit before Safari 16.4) no longer crashes to the error screen when showing a task whose description contains notes — opening such a task, or a Focus/Agenda list containing one, threw "Invalid regular expression: invalid group specifier name". The markdown email-autolink pattern now avoids the regex feature those systems lack, and if markdown rendering ever fails again the description falls back to plain text instead of taking the view down.
- The desktop quick-add suggestion popup now lists every slash command the parser understands: typing `/link`, `/energy`, `/area`, `/archived`, or `/*` (add to today's focus) no longer leaves you without suggestions. `/energy:` goes further and offers the three valid levels, so a stray `/energy:hi` can't end up as task title text. Accepted in the task editor's title field, `/energy:` and `/area:` fill the matching editor fields directly, and `/*` stars the task for today. (#869)
- Arrowing through the desktop suggestion popups now shows which row is selected on every theme. On Nord the highlight color was identical to the popup background, so arrow keys looked like they did nothing (Enter was in fact accepting the invisible selection); Nord's muted surfaces now use the palette's designated highlight shade, and the selected row uses the stronger accent style the search palette already had. On E-ink, where every surface is white, the selected row gets an outline instead. (#869)
- The small window the global quick-add shortcut opens no longer frames the capture card with a thick dark border: the card now fills the window edge to edge. (#869)
- Today's Focus can now be arranged by hand, so the list reads in the order you plan to work it: on desktop, drag rows with the grip handle; on mobile, tap the reorder toggle on the section header to drag tasks and tap Done. Manual order applies while the Focus sort is at its default (an explicit sort still wins), it syncs across devices, and a task keeps its place until it leaves Focus. (#876)
- The mobile Pomodoro timer's Focus/Break phase label now follows the app theme instead of always showing blue/green — on Nord it no longer sticks out as an off-palette blue. (#875)
- The desktop Pomodoro panel is much more compact and can now be folded away: a chevron collapses it to a slim row showing just the time and phase (the timer keeps running), the fold state is remembered, and pressing a task's play button reopens the panel. Linking a task to the timer now uses a searchable picker instead of a long dropdown. (#875)
- Finishing a project's last action now offers to complete the project on the spot: the "What's the next action?" prompt gains a **Complete project** button (desktop and mobile) that archives the project — the same reversible action as the Archive button, completing any remaining tasks. The project details helper text now also says plainly that archiving is how a project is completed. (#874)
- Project lists now show the project's next action according to the order you arranged, not the order tasks were created: the task at the top of the project (after drag-reordering) is the one surfaced, on desktop and mobile. Projects without a manual order keep showing the oldest next action. (#873)
- The mobile quick-capture sheet now shows the full quick-add syntax reference (including `/note:<text>`, `/link:<url>`, and `/energy:<level>`) instead of only the short `@context +Project #tag` line.
- The desktop keyboard-shortcuts help (`?`) now ends with a quick-add syntax table, so every token the parser understands is one keypress away.
- Completing with the real finish time now works from every Done control on mobile: long-press the swipe-revealed **Done** button (with a haptic cue), the **Done** status chip in the task editor, or the status badge in the task preview to pick the completion time — the same affordance the status menu already had. Backdating then switching to another status before saving no longer leaves the picked time on the task. (#835)
- Desktop task date quick actions now respond to **Save** or **Cancel** with one click after using a quick-date shortcut. Picking a day from the mini calendar also stays in the draft until **Save**, instead of updating the task immediately. (#878)
- The desktop Review page top area is now a single row: search, the status filter, and the list controls sit together above the tasks, with sort, grouping, and the details toggle gathered into one **View** popover (no more duplicate Details button or a second half-empty toolbar row). On narrower windows the status chips collapse into a compact Status dropdown.
- Desktop toolbar dropdowns — Sort, Group, and the Review status filter — now open a themed menu that matches the app instead of the operating system's bare native list, which looked broken inside the Review View popover. Keyboard behavior matches a native select: arrows move (skipping disabled entries), Enter/Space picks, Escape closes without changing anything.
- Tasks written through the cloud REST API (including MCP assistants in cloud mode) now follow the same rules as the apps: setting a start date on an Inbox task clarifies it to Next, starring an Inbox task promotes it, demoting a starred task to Inbox removes the star, and stale board/Focus positions are cleared when status changes — previously these API writes skipped the apps' shared update rules, so a dated Inbox task stayed invisible in Focus. Board column position can now also be updated through the API, like the other ordering fields.
- A task that leaves Today's Focus by being completed or archived now releases its manual Focus position, so starring it again later adds it at the end instead of jumping into its old slot.
- Reordering Today's Focus by hand is paused while a search or filter narrows the list (desktop drag handles and the mobile reorder toggle both hide), because rearranging a partial view would scramble the positions of the focused tasks that are filtered out. Clear the filter to reorder the full list.
- The desktop task editor now follows the field order set in Settings → GTD → Task Editor Layout for the Basic section: Status (and any other basic field) can sit above or below the Area/Project/Section row, and the selectors inside that row follow the configured order too — previously that row was always pinned to the top. (#880)
- Input placeholders on desktop are now clearly dimmer than typed text in every theme, and the Area/Project/Section selectors show "No area" / "No project" in muted text when nothing is picked, so example hints like `@home, @work` no longer look like values you entered. (#881)
- Fixed an Android crash when calendar sync created an event on devices whose calendar app returns an unexpected event ID: the app now reports the event as not saved and retries on the next sync instead of crashing.
- Fixed the 1.1.5-rc.1 Android storage regression: the new native SQLite engine failed to initialize because multi-statement schema scripts were split apart mid-trigger, so the app silently ran on its JSON backup — and on large libraries (roughly 2 MB of data or more) that backup could not be read back, leaving the app blank with sync failing. SQLite now initializes correctly, and any tasks or edits made while on rc.1 are merged back into the database on first launch after updating. (#766)
- Keyboard navigation from the sidebar into a task list (`ArrowRight` or `l`) now selects and focuses the current task instead of drawing a focus outline around the entire list, and in Focus the first `ArrowDown` after entering moves exactly one row instead of skipping to the second task. (#890)
- Tasks with a timed due date now offer a "Skip reminders" toggle in the task editor on desktop (the existing mobile switch was renamed to match): the task stays visible in Focus and lists but never fires start/due reminders — useful for recurring chores you track without wanting notifications. (#885)
- Tapping the voice button with no speech-to-text model configured no longer silently closes the add-task dialog: it now stays open and shows a notice pointing to Settings, on desktop and mobile. Note: this replaces the old fallback of saving an untranscribed audio attachment when speech was never set up. (#886)
- The iOS Add task sheet scrolls when the More options are expanded, so the title box and the extra fields are both always reachable. (#887)
- Saving a Focus filter on iOS no longer freezes the app: the save-name prompt used to open as a second overlay on top of the filter sheet, which iOS could silently refuse — the Save button looked dead, the filter was never stored, and navigating away could hang the app. The prompt now appears inside the same sheet and saving works. (#889)
- Quick add can now leave plain-language dates alone: a new "Detect natural-language dates" toggle (Settings → GTD, on by default) controls whether bare phrases like "next week" in a title become dates. Turned off, "Register for the race next week" stays literal text with no due date, while explicit tokens (`/due:next week`, `@context`, `#tag`, …) keep working and being cleaned up exactly as before. The setting syncs across devices. (#742)
- Filter chips can now exclude: tapping a context or tag chip in the Focus/list filters cycles through off → included → excluded (shown struck through in red), so "everything except `#someday-maybe`" is one tap — and mixes freely with included chips ("show `@errands`, hide `#waiting`"). Excluded chips always subtract regardless of the Any / All setting, saved Focus filters remember them, and the state is announced to screen readers on desktop and mobile. (#893)
- Tag filters gain the same Any / All switch contexts got: with two or more tags selected, a "Tag match" control appears next to "Context match", so `#quick` and `#calls` can mean "either" instead of "both". `All` stays the default, and saved Focus filters remember the choice. The "Context match" and "Any" labels are now also properly translated in all languages instead of always showing English. (#892)
- The iOS Add task sheet's picker overlays (Contexts, Project, Area, Priority) no longer hide their bottom rows behind the keyboard: the picker card now lifts above the keyboard the way it already did on Android, and follows the keyboard when the predictive-text bar appears or disappears. (#891)
- Settings are findable by what they're called: the desktop settings search now indexes the actual (translated) titles of every setting on each page, so typing `default`, `project`, or `area` surfaces "Default project flow" and "Default area for new tasks" instead of nothing; mobile Settings gains the same search box at the top of the list; and the time-estimate presets editor arrives on desktop (Settings → GTD), matching the one mobile already had. (#884)
- The mobile Pomodoro end-of-phase alert no longer goes missing while the app is in the background: opening the Focus tab could briefly cancel the pending alert while the saved session was still loading, a re-schedule could be interrupted between removing the old alert and creating its replacement, and on iOS any two notifications created within the same second silently replaced each other (which could also swallow task reminders scheduled in a burst). All three paths are fixed. (#888)
- The mobile app now keeps the self-hosted sync token and WebDAV password in the platform keystore (iOS Keychain / Android Keystore) instead of plain app storage, where they could end up inside device backups. Existing credentials move over automatically the first time the app reads them, and the plaintext copy is scrubbed. Dropbox tokens and AI keys already lived in the keystore; this closes the gap for the remaining two secrets. Reported privately by Alicia Sykes (GHSA-8x25-76x5-jgmr).
- When a Pomodoro phase ends while the desktop window is in the background, Mindwtr now also flashes its taskbar entry (Windows and most Linux desktops; on macOS the Dock icon bounces) and keeps it highlighted until you switch to the window — a cue that survives a buried or missed notification, and works where system notifications can't show at all, such as the portable Windows build on Windows 10. (#528)
- The MCP server (`mindwtr-mcp`) gains an opt-in HTTP mode for remote MCP clients: start it with `--http --http-token <secret>` to serve the same tools at a URL (default `127.0.0.1:8722`, endpoint `POST /mcp`) instead of stdio — this is what lets a self-hosted Mindwtr connect to Gemini's custom apps, which take an MCP server URL. A bearer token of at least 16 characters is mandatory (the server refuses to start without one), works with both the local database and cloud mode, and stdio remains the default, unchanged. (#465)
- The mobile Inbox and Focus screens are calmer. The Process Inbox button is now an outlined tonal button with breathing room below the toolbar instead of a solid blue bar competing with the capture button, and Mind Sweep sits as a neutral control beside sort and filter. Focus section headers drop their horizontal rules and blue all-caps color for quiet gray labels. Task cards everywhere are slimmer: tighter vertical padding, and a checklist's progress shows as a small inline count next to the other metadata instead of a full-width bar.
- The Archive page can now show archived projects, not just tasks: a Tasks | Projects switch at the top filters the list (never mixing the two), with search working in both views. Archived projects can be restored — which also brings back the tasks that were auto-completed when the project was archived — or deleted to the Trash after a confirmation, on desktop and mobile.
- The mobile Done list now shows a proper log by default: most recently completed tasks first, matching desktop. It previously fell back to the generic list order (due date, then creation time), which read as random for finished tasks. (#896)
- The swipe actions on the mobile Archive and Trash screens ("Restore", "Delete") now show in the app language instead of always English.
- The desktop WebDAV "Test Connection" button now works with saved credentials after an app restart: it used to send an empty password (reporting 401 Unauthorized) unless you retyped it, even though sync itself kept working with the stored password. The test now uses the same saved credentials sync uses, and a failed test is recorded in the shared log. (#899)
- WebDAV sync errors now say exactly what failed: the error message and the shared log record the request method, the final file URL, the HTTP status, and the server's response — instead of a bare status code and the configured folder path. A remote read that unexpectedly finds no data is also logged with the URL it hit. This makes server-specific rejections (like Koofr's intermittent 405s) diagnosable from a single shared log. (#898)
- Large libraries on mobile no longer stall for 10–20 seconds after saves: the full-library JSON safety copy (kept for downgrades and recovery) now lands at most once every 5 minutes while you work — and immediately when the app goes to the background — instead of after every save, since the SQLite database is the durable copy. Home-screen and lock-screen widgets keep refreshing right away. A sync that only stamped its status timestamp also no longer re-renders settings-dependent screens or re-arms reminder scheduling, and shared logs now break down backup size, per-step sync timing, and the post-sync refresh so remaining slowness can be pinpointed exactly. (#766)
- Desktop task lists no longer draw a second, input-looking outline around the selected task's title: the row highlight is the single selection marker, and arrow-key navigation moves it cleanly from row to row without leaving a stale highlight behind. The focus outline inside the task editor (inline and popup) is also no longer cut off at the top and left edges. (#860)
- The silent "Add to Mindwtr Inbox" iOS Shortcut now understands quick-add syntax: text like `Buy milk /due:friday @errands #personal +Groceries` is parsed exactly like the in-app capture box (dates, contexts, tags, projects, and the other tokens), honoring the "Clean up quick add text" setting instead of landing verbatim as the task title. The Shortcut's separate Note/Tags/Project fields keep their meaning and win over tokens typed in the text, and a capture that can't be parsed still lands in the Inbox as-is. (#895)
- The desktop Next actions list no longer offers a "N hidden (future start) · Show" reveal: future-start tasks now stay hidden everywhere until their start day, matching Focus on both platforms. The old show/hide setting from earlier versions is ignored, so a synced "Show" preference from an older device can't bring the reveal back. Future-start tasks remain visible in Projects and Search when you want to look ahead. (#900)
- The desktop Completed, Waiting, and Someday lists now have a **Filters** button in their toolbar, so you can narrow the list by context, tag, priority, or time estimate before acting on it — for example, filter Completed down to one context, then Select and bulk-archive the matches. Mobile already had this; the panel and any active filters are shared across the three lists and the summary chip under the title shows what's applied. (#863)
- The calendar's "Projected" preview of a repeat-after-completion task no longer lands on the same day as the task's real next occurrence (an every-3-days task due the 21st showed a duplicate ghost on the 21st): the preview now shows the occurrence after it — the 24th in that example — assuming the task is completed on time. Overdue repeats still project from today, and date-only tasks no longer pick up a stray time in the preview. (#900)
- With iCloud sync, a Focus star set on iPhone/iPad now reaches the Mac. The Mac was reading on/off values from iCloud in a form the sync merge didn't recognize, so it silently turned them off while keeping the rest of the task — the same edit could sync a title change but drop the star. This also affected other on/off values coming from iOS: reminder muting, the calendar's future-occurrence preview toggle, a project's sequential setting, and project Focus. Values set on the Mac always synced to iOS correctly. Tasks that already lost a star this way need it toggled once more on the phone after both apps update. (#902)
- Clicking Save or Cancel right after picking a date in the desktop task editor no longer misses: the quick-date row (Today, Tomorrow, +3 days…) used to collapse at the instant of the click, shrinking the dialog and pulling the button out from under the cursor. The row now stays put until the click lands, then tidies up. On Android, the same quick-date row for the due date is now visible without opening the system date dialog — matching the start and review rows — so one tap sets Tomorrow without touching the picker. (#901)
- Mobile sync no longer claims "No internet connection" when the device is online but Mindwtr's own requests to the sync server keep failing (for example cellular data turned off for the app, or a VPN, DNS filter, or firewall blocking it). The toast now says the sync server couldn't be reached and points at the app's network access; a genuinely offline device keeps the offline message.
- Mobile task lists now show the Time Spent badge next to the other row metadata, matching desktop: with the Pomodoro timer and task linking enabled, a task's recorded time appears in the row instead of only inside the task editor.
- The mobile task editor's top-right button now says **Save** instead of **Done** — it commits your edits, and "Done" read as if it referred to the task status shown right below it.
- Typing a checklist on mobile no longer needs a tap of "+ Add item" between entries: pressing return on a filled item inserts the next empty item right after it and keeps the keyboard up, matching the desktop editor's Enter. Pressing return on an empty item ends editing, and "+ Add item" now puts the cursor straight into the new row.
- Android daily digest, weekly review, and repeating task reminders no longer drift away from their set time. Android treats repeating alarms as inexact and defers them under battery optimization (hours late on strict systems like GrapheneOS, and worse after reboots); each occurrence is now scheduled as an exact alarm that re-arms the next one when it fires, the same mechanism one-time task reminders already used. Times also stay on the wall clock across DST changes and reboots. (#905)
- Switching areas on desktop is now a single keystroke: pressing `1`-`9` (outside any text field) jumps to that area by sidebar order, and `0` clears the area filter — faster than the `Shift+A` chord, which keeps working unchanged. Digits with no matching area do nothing. (#865)
- The German status tag on active projects now reads "Aktiv" (a Turkish "Aktif" had slipped into the German translation). (#904)
- Each project now remembers its own task sort order: pick a sort inside a project (due date for a deadline-driven project, manual order for a packing list) and it stays that way across restarts and syncs to your other devices. Desktop's project sort also catches up with mobile's options — the old Default/Due pair becomes the full sort dropdown (start date, review date, title, oldest, newest). Manual drag-reordering still applies while a project is on Default. (#903)
- Desktop task lists gain a third density, **Condensed**, with rows about 20% shorter than Compact so large lists fit more tasks on screen. Pick it under Settings → Appearance, or use the toolbar Density button (and the toggle-density shortcut), which now cycles Comfortable → Compact → Condensed. (#906)
- Batch capture works on iOS: pasting multi-line text into the quick-add box now offers to create one task per line (matching desktop) instead of one long task, and the quick-add "Import .txt" button no longer fails silently — its "create N tasks?" confirmation used to be swallowed by the closing file picker, so nothing appeared to happen.
- Self-hosted sync tokens shorter than 20 characters no longer create a server that rejects everyone: the cloud server used to load such a token into its allowlist, log success, and then 401 every request (the request path enforces 20-512 characters). The server now refuses to start with a clear message when a configured token can never authenticate, and the desktop and mobile settings forms reject an invalid token when you enter it instead of saving it silently.
- Manual sync no longer reports "Sync completed" while the sync indicator shows a failure: when the upload to the sync server fails and is queued for background retry, the Sync now button on desktop and mobile reports the actual error instead of a green success. Self-hosted setups pointing at the wrong address also get a clearer message — a 405 response now notes the URL may not be a Mindwtr sync server (usually a missing port).
