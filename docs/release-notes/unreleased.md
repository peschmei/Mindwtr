# Mindwtr Unreleased

Changes collected after `v1.1.0` and before the next version tag.

## Highlights

_None yet._

## Full Change List

- Switched mobile local storage to a native SQLite engine (op-sqlite): SQL now executes on a dedicated native thread with a faster call path, the first step of moving storage and sync work off the UI thread on large libraries. Existing data is picked up in place; no migration.
- Finishing or editing a task on mobile no longer waits for the full-library backup snapshot: saves complete after the database write, and the backup and widget refresh land coalesced in the background (bursts of saves now write one backup instead of one each — a single save had spent up to 12 seconds on this with large libraries). Reminder rescheduling also coalesces the several store updates a sync cycle makes into one scan. (#766)
- Friendlier first run for people new to GTD: the welcome screen and the Getting Started project now use plain language, a new starter task shows how to hide task-editor fields you don't use, and an empty Focus view now explains how tasks get there (with capture guidance when the app has no tasks yet).
- The Getting Started project, its tutorial tasks, and the sample inbox items are now created in your app language (17 languages). Re-adding the content after switching languages repairs it in place instead of duplicating it. The first-run welcome screen is translated too.
- The AI assistant now works with newer Claude models (Sonnet 5, Opus 4.7/4.8): requests no longer send a temperature these models reject, and the thinking toggle uses their adaptive thinking mode. Enabling thinking on older Claude models no longer fails either. (#857)
- Android text no longer loses its last letter or line on Android 15/16 at large font sizes ("Off" showing as "Of", settings descriptions ending mid-sentence): the app now pins the pre-Android-15 text drawing behavior so drawn text always matches its measured space. Settings rows and the welcome screen also adapt their layout at large font scales instead of squeezing text into narrow columns. (#632)
- On Android, reopening into Archive or another non-tab screen now keeps the main tabs in the navigation stack, so Back returns to the app instead of closing it. (#842)
- Desktop settings toggles no longer get squashed into ovals when the window is narrow — the switch now keeps its shape and the label text wraps instead. (#858)
- Desktop keyboard shortcuts no longer act on the task list behind an open dialog: with search, quick add, or a prompt open, a stray Enter or action key could complete or open a task in the background list (reported as an accidental Done from inside search). Keys pressed inside the search dialog outside its input now also keep driving the search — arrows and Enter navigate results, and typing returns to the query field.
- Updated bundled dependencies to pick up upstream security fixes (hono, undici, tar, vite, js-yaml, Babel on the JS side; plist/quick-xml, quinn-proto, anyhow in the desktop app).
- The "What's the next action?" prompt (shown after finishing a project's last action) now understands quick-add syntax, so `Chase the reply /waiting %Bob` creates the follow-up as a Waiting For task directly — no need to create it as a next action and edit it afterwards. (#859)
- The Focus toolbar now matches every other list: its buttons and the grouping dropdown share one height and shape, the grouping dropdown carries the same `GROUP` label as elsewhere, and its text is no longer clipped at the bottom. (#861)
- Desktop lists now set a task's status straight from the keyboard: press `s` then a letter — `si` Inbox, `sn` Next, `sw` Waiting, `ss` Someday, `sd` Done, `sa` Archived — with an undo toast after the change. The chord works in all three keybinding styles, and `Insert` jumps to the add-task input (or opens quick add). (#860)
- Desktop can now capture email into the Inbox (Settings → Integrations → Email capture): point it at an IMAP folder with an app password (Gmail, iCloud Mail, Fastmail, or any standard IMAP server) and, while Mindwtr is running, each email in that folder becomes an Inbox task — subject as the title, sender and body as the description. Mail is read-only: nothing is modified, moved, or deleted, the folder is created for you if missing, and already-imported messages are remembered so nothing imports twice. (#35)
- The desktop "switch to Area" chord is now forgiving: keeping Shift held from the `A` into the number (which types `!`, `@`, …) still switches areas, pressing Shift mid-chord no longer cancels it, and with Caps Lock on a plain `a` opens quick add instead of silently arming the chord. The shortcut help sheet now spells the chord as `Shift+A 1-9` so it reads as typed. (#865)
- Adding a task to a project during the weekly review is no longer title-only: the prompt understands the full quick-add syntax (`Chase quote @phone %Bob /waiting`), and a new "Save & edit" button creates the task and opens it in the editor — without leaving the review or losing your place. Works on desktop and mobile.
- The desktop Proxy URL setting now applies to all sync traffic. Self-hosted cloud, WebDAV, and Dropbox connection requests had been going out directly, ignoring the saved proxy (only reachable via `HTTP_PROXY`/`HTTPS_PROXY` env vars as a workaround); they now route through the configured proxy, taking effect on save with no restart. An invalid proxy URL is rejected when saving instead of being silently skipped. (#864)
- Completing a recurring chore no longer clutters Next actions with the occurrence you cannot start yet: a repeat that carries only a due date (a bimonthly "descale the kettle") now stays out of Next actions until it comes due, exactly as it already did in Focus. The deferred occurrences are counted by the "hidden (future start)" notice above the list, with a **Show** toggle to reveal them. The Android daily review defers them the same way. (#867)
- "Repeat after completion" tasks that also pick weekdays (or a day of the month) now honor the **Repeat every** interval. An "every 2 weeks on Sunday" chore completed on a Tuesday used to come back the very next Sunday — and the calendar preview showed the same too-early date — no matter what interval was set; it now returns on the Sunday of the second week after completion. Fixed-schedule repeats are unaffected. (#867)
- Desktop keyboard shortcuts no longer go dead after deleting a task from its open editor (for example Quick Add → Save & edit → Delete): the app was still treating the removed task as being edited and kept suppressing every shortcut until another edit was opened by mouse. (#870)
- The Trash now supports bulk actions on desktop and mobile, completing the set alongside Completed and Archived: tap **Select** to pick individual tasks and projects (or **Select all**), then restore them to the Inbox or delete them permanently in one step — permanent deletion still asks for confirmation. (#863)
- The desktop Add Task dialog is now fully keyboard-driven: `Ctrl+Enter` (`⌘Enter` on macOS) saves and opens the task in the editor, and `Shift+Enter` saves and keeps the dialog open for the next capture — batch entry without touching the mouse (the Save buttons show these shortcuts as tooltips). Tab no longer stops on the dialog backdrop or the `Esc` chip, and the syntax help line now sets the typable tokens (`/start:`, `@context`, `+Project`, …) apart from their `<placeholder>` parts in every language. (#869)
- The weekly review's final screen now ends with a short, calm status summary instead of only a congratulation: whether your Inbox is empty, whether every active project has a next action, and how many Waiting For items have sat untouched for more than two weeks. Each line mirrors what the matching review step showed, and the good/needs-attention rows read at a glance without scores or noise. Works on desktop and mobile. (#871)
- Setting a start date on an Inbox item now clarifies it to Next — the same shortcut starring already took. Deciding *when* you can act on something is a clarify decision, and before this an Inbox item with a start date was invisible in Focus (Focus only lists actionable tasks), which read as the date being ignored. Someday and Waiting items keep their status when dated (a dated someday is a tickler, a dated waiting-for a follow-up reminder), picking a status in the same edit wins, and a date arriving still never changes status on its own. Works everywhere tasks are edited or captured: editors, quick-add `/start:`, review defer, and the MCP/CLI/API surfaces.
- Desktop on older macOS (WebKit before Safari 16.4) no longer crashes to the error screen when showing a task whose description contains notes — opening such a task, or a Focus/Agenda list containing one, threw "Invalid regular expression: invalid group specifier name". The markdown email-autolink pattern now avoids the regex feature those systems lack, and if markdown rendering ever fails again the description falls back to plain text instead of taking the view down.
- The desktop quick-add suggestion popup now lists every slash command the parser understands: typing `/link`, `/energy`, `/area`, `/archived`, or `/*` (add to today's focus) no longer leaves you without suggestions. `/energy:` goes further and offers the three valid levels, so a stray `/energy:hi` can't end up as task title text. Accepted in the task editor's title field, `/energy:` and `/area:` fill the matching editor fields directly, and `/*` stars the task for today. (#869)
- Arrowing through the desktop suggestion popups now shows which row is selected on every theme. On Nord the highlight color was identical to the popup background, so arrow keys looked like they did nothing (Enter was in fact accepting the invisible selection); Nord's muted surfaces now use the palette's designated highlight shade, and the selected row uses the stronger accent style the search palette already had. On E-ink, where every surface is white, the selected row gets an outline instead. (#869)
- The small window the global quick-add shortcut opens no longer frames the capture card with a thick dark border: the card now fills the window edge to edge. (#869)
- Completing with the real finish time now works from every Done control on mobile: long-press the swipe-revealed **Done** button (with a haptic cue), the **Done** status chip in the task editor, or the status badge in the task preview to pick the completion time — the same affordance the status menu already had. Backdating then switching to another status before saving no longer leaves the picked time on the task. (#835)
